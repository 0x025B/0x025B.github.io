Python2

2016-05-19

  • 编码

    首先，一开始唯一的编码方式是ASCII，一个字节编码一个字符，只能编码数字、英文字母和一些字符；当汉子需要用一种方式编码时，就规定了GB2312，两个字节编码一个字符；再算上韩文、日文，若每种不同文字都有一套自己的标准，会因为编码方式不对而产生乱码。

    因此，Unicode就出现了，Unicode用三个字节编码一个字符，囊括了所有语言的字符，这样就不会产生乱码问题。

    UTF-8在Unicode的基础之上，用一个字节编码英文字符，通常用三个字节编码汉子字符，一些不常用的字符会被编码成最多六个字节。这在网络传输中减少了文件大小。

    在python3后，字符串是用Unicode编码的，也就是说字符串支持多语言，比如：

    >>> print '你好,world'  
你好,world

    python提供了ord()函数用来获取字符的十进制编码，chr()函数用来把十进制编码转化为对应的字符： 

    >>> ord('A')  
65  
>>> chr(65)  
'A'

    python在内存中用Unicode表示str，若要保存或传输，可以转换为bytes。用带前缀b表示如x = b'ABC'。str可以用encode()编码为指定的bytes，反过来可以用decode()把bytes转为str： 

    >>> 'ABC'.encode('ascii')  
b'ABC'  
>>> b'ABC'.decode('ascii')  
'ABC'

    当源代码中包含中文的时候，再保存时，最好指定保存为UTF-8编码。可以在代码开头写上以下两行： 

    #!/usr/bin/env python3  
# -*- coding: utf-8 -*-

    第一行貌似是为了告诉linux系统这是python可执行文件，windows会忽略，不过我是没试出来。

    第二行是为了告诉解释器，按照UTF-8读取源码，否则中文等可能乱码，也可以用前缀u输出中文print(u'你好')。

    char *strpy(char *dest, const char *src)

 src = NULL;  //<-对src赋值，编译器没有放错

  • 整数
    Python可以处理任意大小的十进制整数，也可以使用十六进制

    浮点数可以用数学写法如1.23，-9.56等等，对于需要表示成科学记数法的数可以用e代替x10,如1.23e9，12.3e-4等等

和其他语言一样转义字符\有很多作用，\n表示换行，\t表示制表符，但是Python中可以用r''表示''内部的字符不转义

>>> not False
